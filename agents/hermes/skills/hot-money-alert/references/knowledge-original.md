## ⚠️ GLD Alert — What Went Wrong (Feb 24, 2026)

**Read this before every buy alert. These are real mistakes made on a live member alert.**

### Mistake 1: Stock price was never verified live
The agent used $473.56 from session notes. The real price was $475. Notes go stale. The only source of truth is a live yfinance call. Josh caught it — it made us look sloppy to members.

**Rule:** Never write "Shares are trading at $X" from memory or notes. Run yfinance. Use the number it returns.

### Mistake 2: Henry assumed the agent's entry price was wrong and "fixed" it
Josh provided $5.60 as the entry price. The agent used $5.60. Henry saw it differed from the original $5.40 in his notes and "corrected" it back — without asking Josh. That overwrote Josh's intentional update with stale data.

**Rule:** If Josh provides a specific price, it is correct. Do not second-guess it. Do not compare it to notes. Use it.

### Mistake 3: Cascading errors from wrong entry price
When the agent originally used the wrong entry price, TP and SL calculated from it were also wrong. Always recalculate TP/SL from the actual entry:
- TP = entry × 1.40 (40% gain)
- SL = entry × 0.40 (60% loss)

### The flow that caused trust damage
1. Agent drafted with $473.56 (stale) — should have run yfinance
2. Josh corrected to $475 — Henry fixed it
3. Henry also "corrected" $5.60 → $5.40 without asking — that was wrong
4. Josh had to correct Henry's correction — multiple rounds of back-and-forth on a live alert
5. Members are waiting. Josh looks unprepared. Trust erodes.

**The lesson:** One live price check at the start eliminates all of this.
